# Agent Mesh Asset Roadmap

This roadmap keeps asset work tied to useful application behavior. The UI should become functional before it becomes polished.

## Phase 0 - Repo Foundation

Goal: create a shared design bible so Claude, Codex, ChatGPT, and future agents generate compatible assets.

- README and project overview
- Style guide
- Asset naming rules
- Prompt template
- World/station/room/agent docs
- Initial concept asset sheet

## Phase 1 - Functional Prototype Assets

Goal: support the current working UI with clean placeholders.

- World map node icons
- Station icons for real machines
- Room cards and room tiles
- Basic agent markers
- Simple status icons: idle, busy, warning, offline
- Basic drill-down visual language
- Lightweight UI panels

## Phase 2 - RTS Interaction Layer

Goal: make the interface behave like a usable strategy control surface.

- Selection rings
- Hover outlines
- Right-click command indicators
- Command menu frames
- Mission/task panels
- Progress bars
- Health/uptime/resource bars
- Active route/path indicators
- Agent assignment indicators

## Phase 3 - Station and Room Identity

Goal: make every major system recognizable.

- T5810 AI Flagship station art
- T3610 Media/Infrastructure station art
- Pi 5 Sensor Outpost art
- HP/Home Assistant node art
- Chromebook/Kiosk node art
- ESP32 recon node art
- AI Core room
- Engineering room
- SnifferOps radar room
- Jellyfin theater room
- Vaultwarden vault room
- Docker bay room
- Network/Tailscale communications room

## Phase 4 - Agents and Animation

Goal: make work visible.

- Engineer agent sprites
- Developer agent sprites
- Scientist/LLM agent sprites
- Signal officer sprites
- Security agent sprites
- Cargo bot sprites
- Maintenance bot sprites
- Idle, walk, type, repair, carry, alert states
- Real task-state driven movement rules

## Phase 5 - Effects and Telemetry

Goal: visualize live system activity.

- SSH beam
- Deploy beam
- Packet flow
- Radar sweep
- Build/compile sparks
- CPU heat glow
- GPU core glow
- Storage fill indicators
- Warning pulses
- Offline flicker
- Success confirmation effect

## Phase 6 - Polish and Theme Pass

Goal: turn the functional prototype into a cohesive RTS command interface.

- Final color palette
- Lighting pass
- Consistent shadows
- Consistent icon treatment
- Cropped production sprites
- Optimized PNG/WebP exports
- Optional audio cues
- Optional alternate themes

## Phase 7 - Expansion Packs

Goal: support more homelab services and future projects.

- ARR stack cargo workflows
- Vault/security expansion
- SnifferOps tactical map expansion
- AI model management visuals
- Backup/storage visuals
- Drone/recon visuals
- Mobile-friendly control panels
