# Shape Language

Shape language keeps Agent Mesh readable even before final art is polished.

Objects should be identifiable by silhouette, not just color.

## Core Principle

Different object types should have different silhouettes.

The user should be able to tell what something is at a glance.

## Stations

Stations are large world-map objects.

Shape ideas:

```text
AI Flagship: long flagship/server-core silhouette, reactor center
Media Station: blocky cargo/media hub, storage pods
Sensor Outpost: compact dish/antenna shape
Automation Node: utility cube or control node
Kiosk Node: small terminal/display shape
ESP32 Recon Node: tiny drone/beacon shape
Cloud Relay: soft orbital/cloud node silhouette
Backup Station: heavy archive/storage vault shape
```

Station shapes should support status rings around them.

## Rooms

Rooms are modular interior spaces.

Shape ideas:

```text
Bridge: command table / central hologram
AI Core: circular reactor / model core
Engineering: workbench / tool wall
Docker Bay: container racks / crane rails
Jellyfin Theater: screen / media shelves
ARR Cargo Office: crates / conveyor / desk
Vaultwarden Vault: vault door / lock console
SnifferOps Radar: circular radar dish/scope
Communications: network map table / relay consoles
Storage Room: drive racks / storage silos
Build Lab: test bench / terminal row
```

Rooms should have open center or side space for agents.

## Agents

Agents should be small, readable, role-distinct silhouettes.

Shape ideas:

```text
Engineer: tool pack, repair arm, sturdy stance
Developer: terminal pad, slimmer tech silhouette
Scientist: analysis visor, purple data device
Signal Officer: antenna pack, radar scanner
Security Officer: shield/lock motif, heavier shoulders
Cargo Bot: boxy loader bot, cargo arms
Maintenance Bot: compact utility bot
Recon Drone: small hovering sensor shape
Commander: distinct coat/cape/command silhouette later
```

Avoid over-detailing agents. At small size, silhouette matters more than texture.

## UI Panels

UI panels should use:

```text
rectangles with clipped corners
thin border lines
small bracket details
subtle inner glow
blank center area for app text
```

Avoid:

```text
highly irregular frames
huge decorative corners
busy backgrounds behind text
```

## Status Icons

Status icons should use shape plus color.

```text
healthy: circle/check-like mark
busy: rotating ring/dot cluster
warning: triangle/chevron
error: diamond/octagon/cross-like mark
offline: broken ring/slash
blocked: pause/lock-like mark
complete: check/success pulse
```

## Effects

Effects should have readable direction/meaning.

```text
SSH beam: straight/segmented connection line
Deploy beam: transfer stream with moving packets
Packet flow: small dots along path
Radar sweep: arc or rotating wedge
Warning pulse: expanding ring or corner flash
Success pulse: short outward glow
```

## Props

Props should be functional shapes.

```text
terminal: angled console or screen block
server rack: vertical rectangular cabinet
gpu rack: heavier rack with glowing core slots
vault door: circular or heavy rectangular lock door
radar screen: circular scope
crate: simple box/cargo shape
antenna: mast/dish shape
repair bench: table with tool silhouette
```

## Readability Rule

If a silhouette only makes sense with a label, it needs a stronger shape.
